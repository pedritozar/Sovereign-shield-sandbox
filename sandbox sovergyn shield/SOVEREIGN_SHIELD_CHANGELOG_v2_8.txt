   Uso: Posición de la ISS en tiempo real.
   Mecánica: ISS sobre la base → bonus de radar temporal. Sprint 5.

   Uso: GIFs animados para el Feed de Prensa dinámico.

=======================================================================
 SISTEMA DE TIEMPO ASINCRÓNICO (IKARIAM ENGINE)
=======================================================================

PRINCIPIO: El tiempo corre aunque nadie esté conectado.

MECÁNICA Firebase Timestamps:
  Al ordenar una obra → Firebase guarda:
    { timestamp_inicio: [hora_real], duracion_segundos: 7200 }
  Al conectarse → JS compara hora del servidor vs timestamp guardado.
  Si pasaron las 2 horas → obra completada sola, recursos actualizados.

MODO SELECCIONADO: Turnos por Trimestre (Opción 1).
  El juego avanza solo cuando AMBOS confirman sus acciones.
  PEI carga CSV → primo construye → ambos aprueban → avanza trimestre.
  VENTAJA: Compatible con horarios de facultad y trabajo de ambos.

MODO DE TIEMPO (selector en UI):
  - Turnos: consenso trimestral (DEFAULT)
  - Acelerado: 1 día real = 1 mes virtual (Firebase timestamp)
  - Libre: Play/Pause manual centralizado en Firebase

MOTOR DE TIEMPO HISTÓRICO (Sprint 7 — nuevo):
  Contador JS separado: 1 Hora Real = 1 Año Virtual del Sandbox.
  Activa triggers automáticos locales sin consumir API.
  Ver sección MECÁNICAS DE JUEGO PROFUNDO — Motor de Eventos.

=======================================================================
 OBSTÁCULOS TÉCNICOS CONOCIDOS — REGLAS PARA LAS IAs
=======================================================================

1. TIPOS DE VARIABLES (JavaScript no tipado):
   PapaParse lee strings. SIEMPRE parseFloat() o Number() antes
   de operar. Error típico: "1500" + 1 = "15001".

2. LEAFLET ASÍNCRONO:
   SIEMPRE validar: if (typeof L === 'undefined') esperar 500ms.
   Nunca inicializar el mapa antes de que el CDN haya cargado.

3. FIREBASE ESCRITURA MASIVA:
   Animaciones corren LOCAL (CSS/Canvas/RequestAnimationFrame).
   Firebase recibe SOLO el resultado final (Éxito/Fallo/Costo).

4. API KEY SEGURIDAD:
   NUNCA en código fuente ni en GitHub.
   SIEMPRE: localStorage.getItem('ss_anthropic_key').

5. RACE CONDITIONS EN TESORERÍA:
   SIEMPRE usar transaction() para debitar/acreditar tesorería.
   Nunca usar set() directo para valores financieros compartidos.

6. ANTI-SPAM API:
   Cooldown de 15 segundos en botón de crisis geopolítica.
   Barra de progreso visual durante el cooldown.
